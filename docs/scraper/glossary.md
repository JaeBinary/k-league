# 용어 사전

스크래퍼 문서에서 사용되는 주요 용어를 정의합니다.

## 일반 용어

### 스크래퍼 (Scraper)

웹사이트에서 데이터를 자동으로 추출하는 프로그램. 웹 크롤러, 웹 스파이더라고도 합니다.

### 파싱 (Parsing)

텍스트 데이터(HTML, JSON 등)를 분석하여 구조화된 데이터로 변환하는 과정입니다.

### 크롤링 (Crawling)

웹 페이지를 탐색하며 데이터를 수집하는 과정입니다. 여러 페이지를 순회하며 정보를 수집합니다.

---

## 기술 용어

### BeautifulSoup

Python의 HTML/XML 파싱 라이브러리. CSS 선택자나 XPath로 요소를 찾아 텍스트나 속성을 추출합니다.

### Selenium

웹 브라우저를 자동화하는 도구. JavaScript로 렌더링되는 동적 페이지를 처리할 수 있습니다.

### WebDriver

Selenium에서 브라우저를 제어하는 인터페이스. ChromeDriver, GeckoDriver 등이 있습니다.

### ChromeDriver

Google Chrome 브라우저를 Selenium으로 제어하기 위한 드라이버 프로그램입니다.

### CSS Selector

HTML 요소를 선택하는 패턴. 예: `div.className`, `#elementId`, `[attribute=value]`

### XPath

XML/HTML 문서에서 노드를 선택하는 경로 표현식. 예: `//div[@class='name']`

## 데이터 용어

### 스키마 (Schema)

데이터의 구조와 각 필드의 타입을 정의한 것입니다.

### 딕셔너리 (Dictionary)

Python의 키-값 쌍 자료구조. 경기 데이터는 딕셔너리 형태로 반환됩니다.

```python
{
    'HomeTeam': '울산',
    'AwayTeam': '포항',
    'Audience_Qty': '15234'
}
```

### DataFrame

pandas 라이브러리의 2차원 테이블 구조. 딕셔너리 리스트를 DataFrame으로 변환하여 분석합니다.

### ISO 8601

날짜와 시간을 표현하는 국제 표준 형식. 예: `2025-02-15 14:00:00`

## 처리 방식 용어

### 순차 처리 (Sequential Processing)

작업을 하나씩 순서대로 처리하는 방식. 안정적이지만 속도가 느립니다.

### 병렬 처리 (Parallel Processing)

여러 작업을 동시에 처리하는 방식. 속도가 빠르지만 리소스를 많이 사용합니다.

### 워커 (Worker)

병렬 처리에서 개별 작업을 담당하는 스레드 또는 프로세스입니다.

### ThreadPoolExecutor

Python의 동시성 라이브러리. 스레드 풀을 관리하여 병렬 작업을 실행합니다.

## 축구 용어

### 시즌 (Season)

한 해의 리그 대회 기간. 보통 2월~11월 (K리그), 2월~12월 (J리그)입니다.

### 라운드 (Round)

리그 대회의 경기 주차. K리그1 기준 38라운드, J리그1 기준 38라운드입니다.

### 홈팀 (Home Team)

해당 경기장을 연고로 하는 팀입니다.

### 어웨이팀 (Away Team)

상대방 경기장에서 경기하는 팀입니다.

### 트래킹 데이터 (Tracking Data)

선수들의 이동을 추적하여 수집한 데이터. 주행거리, 스프린트 횟수 등을 포함합니다.

### 주행거리 (Distance)

경기 중 선수들이 이동한 총 거리 (단위: km). 팀 전체의 합계로 표시됩니다.

### 스프린트 (Sprint)

시속 25km 이상으로 달린 횟수입니다.

### 점유율 (Possession)

경기 중 한 팀이 공을 소유한 시간의 비율 (%). 홈팀 55%, 어웨이팀 45%처럼 표시됩니다.

### 슈팅 시도 (Attempts)

골대를 향해 슈팅을 시도한 총 횟수입니다.

### 유효 슈팅 (On Target)

골대 범위 내로 향한 슈팅 횟수. 골키퍼가 막지 않으면 골로 이어지는 슈팅입니다.

### 승점 (Points)

리그 순위를 결정하는 점수. 승리 시 3점, 무승부 시 1점, 패배 시 0점입니다.

## 리그 명칭

### K리그1

한국 프로축구 1부 리그. 12개 팀이 참가하며 시즌 총 228경기입니다.

### K리그2

한국 프로축구 2부 리그. 팀 수에 따라 경기 수가 달라집니다.

### J리그1 (J1リーグ)

일본 프로축구 1부 리그. 20개 팀이 참가하며 시즌 총 380경기입니다.

### J리그2 (J2リーグ)

일본 프로축구 2부 리그. 22개 팀이 참가합니다.

### J리그3 (J3リーグ)

일본 프로축구 3부 리그.

### 승강 플레이오프 (Promotion Playoff)

1부와 2부 리그 간 승격/강등을 결정하는 추가 경기입니다.

## URL 및 API 용어

### meetSeq

K리그 API에서 리그를 식별하는 파라미터. 1=K리그1, 2=K리그2, 3=승강PO

### gameId

K리그 API에서 경기를 식별하는 파라미터. 시즌 내 1부터 순차 할당됩니다.

### league_category

J리그 API에서 리그를 식별하는 파라미터. j1, j2, j3, playoff, 2playoff

### matchRecord.do

K리그 공식 API 엔드포인트. 경기의 기본 통계 데이터(점유율, 슈팅, 파울, 카드 등)를 제공합니다.

### possession.do

K리그 공식 API 엔드포인트. 경기의 15분 단위 시간대별 점유율 데이터를 제공합니다.

## 오류 용어

### TimeoutException

지정된 시간 내에 작업이 완료되지 않아 발생하는 예외입니다.

### NoSuchElementException

페이지에서 찾으려는 HTML 요소가 존재하지 않을 때 발생하는 예외입니다.

### SessionNotCreatedException

WebDriver 세션 생성에 실패할 때 발생하는 예외입니다.

### ConnectionError

네트워크 연결에 실패할 때 발생하는 예외입니다.
